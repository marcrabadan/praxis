Closes PSDOS gaps #3 (architecture validation), #4 (front-half states), upstream
gate criteria, and failure-protocol transitions in feature-development.

- States: added product-definition (PO-owned: MVP scope, prioritised+traced
  requirements, success metrics) and release-candidate (proven-correct vs
  decided-to-ship). New chain: discovery → research → product-definition → spec
  → experience → plan → tasks → build → verify → release-candidate → release.
- Criteria-checked gates: new manifest block `gateCriteria` turns HITL approval
  tokens (approved-discovery / approved-product-definition / approved-spec /
  architecture-validated / release-candidate-ready) into explicit checklists.
  architecture-validated gives architecture its own pass/fail (NFRs, modularity,
  ADR-for-unfamiliar, security-relevant design) via plans/architecture-review.md.
- Failure protocol: new `transitions.onGateFailure` maps a failed gate to its
  rework state (root-cause → return → revalidate); bypassing is a stop condition.
- Schema + validator extended (_check_gate_criteria, _check_transitions) with
  5 new unit tests. Templates added: product-definition.md, architecture-review.md,
  release-candidate.md.

Scope: feature-development only (bug-fix/refinement unchanged, by design).
Validation: validate-harness OK; 105 tests pass; traceability OK.
