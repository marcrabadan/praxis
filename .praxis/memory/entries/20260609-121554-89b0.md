Closes the highest-leverage PSDOS gap: a feature could previously reach `release`
without ever passing a security or performance gate (both were opt-in expert
routing via /security and /review-changes, not gated states).

Decision:
- G-security is MANDATORY (no `conditional` flag, mirroring G-acceptance). Every
  feature records a security review; each high/critical finding is fixed or
  carries an approved risk-acceptance decision. Faithful to PSDOS's
  non-skippable Security Gate. A no-risk change still records the determination
  (evidence), not a silent skip.
- G-performance is CONDITIONAL (mirrors G-visual), appliesTo runtime-bearing
  surfaces [screen, flow, api, job, data, integration]; pure cli/library records
  n/a with reason. Avoids overbuild (AGENTS.md) while honoring PERFORMANCE_REVIEW.

Both added to loops.verify.gates and loops.verify.predicate; a new stop condition
blocks release on an unaddressed high/critical security finding.

Scope: feature-development only. bug-fix and refinement (lighter lifecycles) are
deliberately excluded for now — extend on evidence per promotion-policy.

Files: workflows/feature-development.workflow.json (gateCatalog, predicate, gates,
stopConditions); systems/feature-development/feature-development.md (doctrine);
projects/_template/specs/_template/reports/verify/report.md (gate rows).

Validation: make validate-harness OK; tools unittest 39 passed; JSON well-formed.

Traceability: PSDOS Security Gate + SECURITY_REVIEW/PERFORMANCE_REVIEW states ->
G-security/G-performance -> verify report rows.
