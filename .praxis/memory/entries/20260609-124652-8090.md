(proactive continuous learning).

- Validation Orchestrator: new Tier-2 skill .claude/skills/validation-orchestrator

  halt workflow progression; adjudicates each gate against its gateCriteria,
  reads the verify report (no self-certification, U-8), enforces stop conditions,
  refuses on pending decisions, and returns a closed-set verdict
  advance|block|escalate. references/authority.md carries the gate-by-gate
  checklist + escalation protocol. Deliberately ONE agent (not all four PSDOS
  orchestrators) to stay faithful to praxis's thin-command design; the other
  three remain the existing /new-feature etc. orchestration.
- Pattern miner: tools/patterns.py + `make patterns`. The proactive half of
  continuous learning — sweeps the memory ledger and stop-condition run logs for
  recurring tags/sources/types/stop-conditions and surfaces them as human-gated
  promotion candidates (routed via promote.py / skill-learner). Read-only; never
  mutates. 8 unit tests.

Plugin symlinks added; SKILLS.md regenerated; integrations correctly exclude the
orchestrator (not an SDLC persona). validate-all + catalog-check + integrations-check OK.
