User decision (3 ejes): alcance GLOBAL (praxis + consumidores), AUTO-BOOTSTRAP
cuando no hay proyecto, y harness OBLIGATORIO (sin válvula de opt-out).

Revierte la doctrina previa "harness is opt-in / non-harness repos unaffected"
(documentada en AGENTS/README/docs). Ahora harness es el modo por defecto y único.

Implementación:
- tools/ensure_harness.py (nuevo): bootstrap idempotente, stdlib. Deriva projectId
  del nombre del repo. Harness repo → central, projects/<id>/ + fila en el index.
  Consumidor → local, .praxis/project/. No-op si ya inicializado. make harness-init.
- 4 comandos (new-feature/fix-bug/refine/review-changes): quitada la rama
  "if not in harness, behave as before"; ahora llaman ensure_harness primero.
- Stop conditions acotadas: config AUSENTE ya no es hard-block (se bootstrappea);
  solo config PRESENTE-pero-rota (malformada o projectId que no resuelve) para.
- Doctrina/docs: AGENTS.md, rules/source-of-truth.md, rules/stop-conditions.md,
  README, docs/harness-mode.md, docs/index.html → "always on".
- El propio repo praxis ahora tiene .praxis/config.json + projects/praxis/.
- Test: tools/test_ensure_harness.py (8 casos).

Trade-off aceptado: rompe backward-compat para repos consumidores que antes
corrían en modo no-harness sin config. Es intencional (decisión del usuario).
Validación: 113 tests, validate-harness/all, catalog/integrations/traceability OK.
