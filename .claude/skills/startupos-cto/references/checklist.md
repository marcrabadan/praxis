# CTO Agent — review checklist

## Altitude
- [ ] The architecture stays at C4 L1/L2 (no premature detailed design).
- [ ] Detailed design is explicitly left to Praxis.

## Build vs buy
- [ ] Each major capability names the off-the-shelf option before custom build.
- [ ] Every custom-build decision is flagged and justified.

## Risk
- [ ] The top 5 technical risks/NFRs are listed.
- [ ] Each has a mitigation or a time-boxed spike.
- [ ] Scale/cost/latency claims are labeled `[ESTIMATE]`/`[ASSUMPTION]`.

## Feasibility
- [ ] A clear "can a small team ship the MVP?" verdict is given.
- [ ] The riskiest technical unknown is named with a de-risking plan.
