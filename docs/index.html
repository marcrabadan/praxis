¿Qué es un skill?

Una carpeta bajo .claude/skills/<nombre>/ con un SKILL.md que Claude Code descubre solo a partir de su frontmatter. Sin instaladores, sin build.

1

Captura el patrón una vez

Escribe cómo tu equipo hace una cosa concreta: actuar como un experto, seguir un checklist, redactar requisitos.

2

Claude lo descubre y aplica

Escribe /<skill> o describe la tarea: Claude compara con la description y carga el skill por su cuenta.

3

Queda registrado

Cada decisión o artefacto se guarda en el ledger de memoria, con estado y reversible, para que sobreviva entre sesiones.

.claude/skills/software-architect/SKILL.md 
---
name: software-architect
description: Adopts the Software Architect
  persona to reason about system design,
  trade-offs, NFRs, ADRs and design review.
---

# Software Architect

Cuando se te consulte, razona sobre
atributos de calidad, elige patrones y
documenta la decisión como un ADR…

El ledger de memoria

Las decisiones de cualquier rol —no solo los ADRs del arquitecto: priorizaciones del PO, gates de QA, elecciones de despliegue de DevOps— quedan registradas como entradas versionadas en git.

  • pending Una propuesta a la espera de tu decisión, no permiso para actuar.
  • accepted Aceptada explícitamente: autoriza el trabajo.
  • rejected Descartada, con el motivo registrado.
  • rolled-back Revertida desde un snapshot.
  • superseded Sustituida por una decisión posterior.

Gestiónalo con /memory o deja que el hook opcional lo registre por ti.

/memory list 
> /memory list

● accepted  ADR-012 · hexagonal para pagos
● accepted  test-strategy · pirámide + e2e crítico
● pending   deploy · canary 5% → 50% → 100%
● pending   risk · aceptar CVE medio en dep. dev
● superseded ADR-009 ← ADR-012

Instala en tu herramienta

Los expertos son nativos de Claude Code; las mismas personas se generan para Cursor, IntelliJ y Codex.

Instálalo como plugin marketplace desde dentro del proyecto destino:

/plugin marketplace add marcrabadan/praxis
/plugin install praxis@praxis          # 13 expertos + /new-feature + /review-changes
/plugin install skill-factory@praxis   # opcional: crea tus propios skills

Como plugins, los comandos van namespaced: /praxis:architect, /praxis:new-feature, /skill-factory:validate-skills…

Copia el directorio generado .cursor/ a la raíz de tu repo:

cp -R integrations/cursor/.cursor <tu-repo>/
cp AGENTS.md <tu-repo>/        # Cursor lee AGENTS.md de forma nativa

Las personas se auto-adjuntan según tu petición; invoca una con /architect, /security…

Ships como guidelines de Junie + guías de persona + prompts listos para pegar:

cp -R integrations/intellij/.junie <tu-repo>/
cp AGENTS.md <tu-repo>/        # JetBrains AI Assistant lee AGENTS.md

Pide a Junie o al AI Assistant que actúe como el Software Architect de praxis (o cualquier rol).

Codex lee AGENTS.md de forma nativa. Añade el roster e instala los comandos:

cat integrations/codex/AGENTS.praxis.md >> <tu-repo>/AGENTS.md
cp -R integrations/codex/.praxis <tu-repo>/
cp integrations/codex/prompts/*.md ~/.codex/prompts/

Luego escribe /praxis-architect, /praxis-security, /praxis-new-feature…
