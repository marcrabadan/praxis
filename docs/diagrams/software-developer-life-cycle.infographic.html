Verify ⟳ Bounded loop

Prove every acceptance criterion against a typed gate catalog. Iterates to its predicate or escalates — it never spins. Self-certification is forbidden.

OWNERQA Engineer+Security EngineerG-security · mandatoryG-performance · conditional
↺ on fail → route back to  Build  (bounded: max iterations, then escalate)

Release Candidate Proven correct

Verify passed with sign-off; release notes drafted; no open high/critical security finding. Separates proven-correct from decided-to-ship.

OWNERDevOps Engineer→ release-candidate.md
🛡️ Gate 4 — Release approved. The human decision to ship the proven candidate. HITL
🚀11

Release Ship

Publish with release notes and a rollback plan; record acceptance met and what shipped.

OWNERDevOps Engineer→ release-notes.md

Deploy Optional · config-gated

Runs only when .praxis/config.json declares a deploy target (else it records a skip). Provisions infra with Terraform and ships to Kubernetes / AWS / GCP / Azure through an MCP server — with pre-apply guardrails, supply-chain checks and an SLO-gated rollout.
